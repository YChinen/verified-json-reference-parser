# セキュリティポリシー

## サポート対象バージョン

本プロジェクトは現在も開発中です。
現時点では **`main` ブランチの最新版のみ**をサポート対象としています。

セキュリティ修正は原則として最新の開発バージョンに適用されます。

---

## 脆弱性の報告方法

セキュリティ上の問題（脆弱性）を発見した場合は、**公開Issueではなく非公開で報告**してください。

推奨される報告方法:

* GitHub Security Advisory（Private vulnerability report）

**脆弱性を公開Issueとして投稿しないようにしてください。**

報告の際は、可能であれば以下の情報を含めてください:

* 問題を引き起こす入力（JSON ドキュメントおよび JSON Pointer）
* 期待される動作
* 実際の動作
* 想定される影響（例: DoS、誤った解決結果など）

報告を受け取り次第、可能な限り速やかに確認および調査を行います。

---

## 対象範囲（Scope）

本ライブラリは **JSON Pointer 文字列および JSON ドキュメント**を処理します。
これらの入力は信頼できない外部入力である可能性を前提としています。

セキュリティ上の観点で主に関係する処理は以下です。

* JSON Pointer のパース
* JSON Pointer の評価（JSONドキュメントに対する解決）
* ローカル JSON Reference の解決
* 不正入力または悪意のある入力の処理

本ライブラリは **コードの実行を行いません**。
また、ファイルシステム・ネットワークなどの外部リソースにはアクセスしません。

---

## セキュリティ上の考慮事項

### 入力サイズ

非常に長い JSON Pointer 文字列や、深くネストした JSON ドキュメントは、
メモリ使用量や処理時間の増加につながる可能性があります。

信頼できない入力を処理する場合は、
アプリケーション側で適切な入力サイズ制限を設けることを推奨します。

---

### Prototype Pollution

本ライブラリはオブジェクトを書き換える操作を提供していません。

ポインタ評価の際には、継承されたプロパティを辿らないよう
明示的なプロパティチェック（例: `hasOwnProperty`）を使用しています。

ただし、アプリケーション側でポインタトークンを利用して
オブジェクトを書き換える処理を実装する場合は、次のようなキーに注意してください。

* `__proto__`
* `constructor`
* `prototype`

---

### 仕様の厳格な取り扱い

本実装は **RFC 6901 (JSON Pointer)** の仕様に基づいています。

不正なエスケープシーケンスや無効なポインタ形式は、
曖昧な解釈を行うのではなくエラーとして扱われます。

この設計は、ポインタ解決の曖昧さを防ぐことを目的としています。

---

## セキュリティ特性

本ライブラリは以下の特性を持つよう設計されています。

* セマンティックコアは例外を投げず、`Result` 型による明示的なエラー処理を行います
* 不正入力は明示的なエラー値として扱われます
* JSON Pointer のエスケープ／アンエスケープの意味論が形式的に定義されています

JSON Pointer の一部の基本的な不変条件については、
**Rocq による機械検証（machine-checked proof）**が行われています。

これには、ポインタのフォーマットとパースのラウンドトリップ特性などが含まれます。

また、開発用の検証レイヤーには、Phase 1 の具体ケースに対する Z3 ベースのオラクルと差分検証も含まれます。

---

## 脆弱性公開ポリシー

脆弱性が確認された場合、以下の手順で対応します。

1. 問題を再現し、原因を調査します
2. 修正を実装し、テストを追加します
3. 必要に応じてセキュリティアドバイザリを公開します
4. 修正版をリリースします

責任ある脆弱性報告（Responsible Disclosure）に感謝します。
