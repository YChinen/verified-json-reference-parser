# 形式仕様 (Formal Specification)

## 概要

本ドキュメントは、本プロジェクトにおける **形式仕様および Rocq による機械検証された証明**について説明するものです。

形式仕様の目的は、JSON Pointer 実装における重要な意味論的性質を明確に定義し、証明支援系を用いてそれらを検証することです。

本プロジェクトでは **Rocq** を使用し、
**RFC 6901 (JSON Pointer)** に基づく意味論の一部の不変条件を形式化し、機械検証を行っています。
加えて、Phase 1 の具体ケースに対する差分検証には Z3 ベースの検証ツールを用いていますが、こちらは Rocq の証明成果物そのものではありません。

形式検証は、実装バグが発生した場合に誤ったポインタ解決につながる可能性がある重要な部分に適用されています。

---

## 形式検証の対象範囲

本プロジェクトにおける形式検証は、現在のところ **ポインタトークンおよびポインタ表現のコア意味論**に焦点を当てています。

形式化されている主な要素は以下です。

* ポインタトークンのエスケープおよびアンエスケープ
* ポインタのパース
* ポインタのフォーマット
* パースとフォーマットの往復特性（round-trip invariants）

これらの性質を保証することで、内部ポインタ表現が一貫して扱われることを目指しています。

---

## 形式モデル

形式モデルでは、JSON Pointer の概念を Rocq のデータ構造で表現します。

### トークンモデル

ポインタトークンは文字列としてモデル化されます。

基本的な操作は以下の2つです。

* `escapeToken : string → string`
* `unescapeToken : string → Result string`

これらは RFC 6901 で定義されているエスケープ規則に対応します。

* `"~"` → `"~0"`
* `"/"` → `"~1"`

---

### ポインタ表現

ポインタはトークン列として表現されます。

```id="k5v5gk"
Pointer := list string
```

空ポインタ (`""`) は空リストに対応します。

---

### パースとフォーマット

文字列表現と内部表現の変換を行う関数として、以下を定義しています。

```id="w2rjqq"
parsePointer  : string → Result Pointer
formatPointer : Pointer → string
```

これらは TypeScript 実装におけるパースおよびフォーマット処理に対応しています。

---

## 証明済みの性質

現在、Rocq によって以下の性質が機械検証されています。

### トークンの往復特性

トークンをエスケープしてからアンエスケープすると、元のトークンが得られます。

```id="x2qg4v"
unescapeToken (escapeToken t) = Ok t
```

この性質により、トークンエスケープが可逆的であることが保証されます。

---

### ポインタの往復特性

ポインタをフォーマットしてからパースすると、元のポインタが得られます。

```id="1on64f"
parsePointer (formatPointer p) = Ok p
```

この性質により、テキスト表現と内部表現の整合性が保証されます。

---

## 実装との対応関係

Rocq による形式モデルは、TypeScript 実装のセマンティックコアに対応しています。

本プロジェクトでは、以下の関係を維持することを目標としています。

```id="zws4z6"
TypeScript 実装
        ↓
Rocq による形式仕様
        ↓
機械検証された証明
```

形式モデルは、実装が満たすべき意味論の参照仕様として機能します。

---

## 検証方針

本プロジェクトの形式検証は、実装全体を証明するのではなく、
**重要な意味論的性質を小さく検証する**方針を採用しています。

主な原則は以下の通りです。

* 重要な意味論的不変条件を検証する
* 形式モデルを単純かつ保守しやすく保つ
* 証明とテストを組み合わせる

このアプローチにより、実用的な開発フローと正当性保証のバランスを取っています。

---

## 制限事項

現在の形式仕様では、以下の内容は対象としていません。

* Rocq による JSON ポインタ評価アルゴリズム全体の証明
* JavaScript 実行環境の正当性
* パフォーマンス特性
* メモリ安全性
* Z3 オラクル実装自体の形式証明

本形式仕様は、ポインタのコア意味論の正しさに焦点を当てています。

---

## 今後の拡張

将来的に、以下の領域を形式化する可能性があります。

* JSON 値モデルの定義
* Rocq によるポインタ評価 (`get`) のより深い検証
* 配列インデックス処理の形式化
* Rocq モデルと TypeScript 実装の整合性証明

機能拡張に応じて、形式仕様も段階的に拡張される予定です。
