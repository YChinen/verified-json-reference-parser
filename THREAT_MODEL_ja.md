# 脅威モデル (Threat Model)

## 概要

本ドキュメントは、本プロジェクトにおける脅威モデルを説明するものです。

本ライブラリは **JSON Pointer (RFC 6901)** のパースおよび評価と、Phase 1 のローカル JSON Reference 解決を実装しています。
JSON Pointer 文字列、ローカル参照文字列、および JSON ドキュメントは **信頼できない入力**として扱われる可能性があります。

本ドキュメントの目的は、以下を明確にすることです。

* 本プロジェクトが保護対象とする資産
* 信頼しない入力
* ライブラリの攻撃面
* 想定される脅威
* それらに対する対策

---

## 保護対象（Assets）

本プロジェクトが主に保護対象とするのは次の特性です。

**JSON Pointer の意味論が正しく、予測可能に評価されること**

具体的には以下を保証することを目標としています。

* JSON Pointer が RFC 6901 に従って解析される
* ポインタ評価が決定的に行われる
* 不正な入力が明示的にエラーとして扱われる

ポインタ解決の誤りは、設定処理・データ検証・ポリシー評価などにおいて誤動作を引き起こす可能性があります。

---

## 信頼モデル（Trust Model）

### 信頼する要素

以下は正しく動作するものとして前提としています。

* JavaScript 実行環境
* ホスト環境の標準ライブラリ
* 本ライブラリの TypeScript 実装

### 信頼しない入力

以下の入力は **信頼できない入力**として扱います。

* JSON Pointer 文字列
* ローカル参照文字列（`#`, `#/...` および不正形式）
* JSON Pointer 評価対象となる JSON ドキュメント

これらの入力は、不正または悪意のある形式で与えられる可能性があります。

---

## 攻撃面（Attack Surface）

本ライブラリの主な攻撃面は以下の通りです。

### JSON Pointer のパース

外部から与えられたポインタ文字列を解析します。
不正なエスケープシーケンスや特殊な入力構造が、解析ロジックの誤動作を誘発する可能性があります。

### ポインタ評価

パースされたトークンを用いて JSON ドキュメントを走査します。
評価ロジックの不具合により、誤った解決結果が返される可能性があります。

### ローカル参照解決

ローカル参照解決では fragment の分離と pointer 評価へのディスパッチを行います。
対応形式と未対応形式の分類を誤ると、誤った解決結果を返す可能性があります。

---

## 想定される脅威

### サービス拒否（DoS）

悪意ある入力により、過剰なリソース消費を引き起こす可能性があります。

例:

* 極端に長い JSON Pointer
* 非常に多くのポインタセグメント
* 深くネストした JSON 構造

信頼できない入力を扱う場合、アプリケーション側で入力サイズ制限を設けることが推奨されます。

---

### Prototype Pollution

JavaScript オブジェクトのプロトタイプ継承に起因するリスクが存在します。

本ライブラリでは、オブジェクトプロパティの解決時に
明示的なプロパティ存在チェックを行うことで、継承プロパティの走査を避けています。

ただし、アプリケーション側でポインタトークンを用いて
オブジェクトを書き換える処理を実装する場合は、以下のキーに注意してください。

* `__proto__`
* `constructor`
* `prototype`

---

### 仕様解釈の曖昧性

JSON Pointer の仕様解釈の違いにより、実装間で挙動が異なる可能性があります。

本実装では **RFC 6901** に厳密に従うことを目標としています。

不正なエスケープや無効なポインタ形式は、
曖昧に解釈するのではなくエラーとして扱います。

---

### 実装バグ

ポインタのパースや評価ロジックにバグが存在した場合、
誤ったポインタ解決結果を返す可能性があります。

これは、ポインタ解決に依存するアプリケーションの動作に影響を与える可能性があります。

---

## セキュリティ対策（Security Measures）

本プロジェクトでは、以下の対策を採用しています。

### 明示的なエラー処理

セマンティックコアでは例外を使用せず、
成功または失敗を `Result` 型で明示的に表現しています。

これにより、不正入力による予期しない例外発生を防ぎます。

---

### 厳格なパース

不正なポインタ入力は寛容に解釈せず、
明示的にエラーとして扱います。

これにより曖昧な解釈を防ぎます。

---

### プロパティ走査の安全化

オブジェクト走査では、継承プロパティを辿らないよう
明示的なプロパティ存在チェックを行っています。

---

### 機械検証による不変条件

JSON Pointer のコア部分については、
**Rocq を用いた機械検証（machine-checked proof）** を行っています。

これには以下の性質が含まれます。

* トークンのエスケープとアンエスケープの往復性
* ポインタのフォーマットとパースの往復性

これにより、重要な意味論のバグの可能性を低減しています。

---

### 追加検証

本プロジェクトでは以下の検証手法も利用しています。

* property-based testing
* differential testing
* 反例探索（counterexample search）
* Phase 1 の具体ケースに対する Z3 ベースのオラクル

これらは境界ケースの発見に役立ちます。

---

## 非対象（Non-goals）

本ライブラリは以下を対象としていません。

* 実行環境自体のセキュリティ問題
* 本ライブラリを利用するアプリケーションの脆弱性
* ポインタトークンを用いたオブジェクト更新処理

アプリケーション側では、用途に応じた入力検証およびアクセス制御を実装する必要があります。

---

## 今後の検討事項

今後の機能拡張に伴い、以下の領域について追加のセキュリティ検討が必要になる可能性があります。

* ポインタを用いた更新 API
* JSON Reference や JSON Schema との統合
* 拡張ポインタ仕様のサポート

新機能の追加に応じて、本脅威モデルは更新される予定です。
